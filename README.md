# AIditor

AIditor 是一个面向 WordPress 后台的通用 AI 采集插件。

你只需要提供列表页或详情页 URL，并用自然语言描述希望采集的字段，AIditor 就可以分析页面、发现详情页、抽取内容、按需调用 AI 重写，并以队列任务的方式批量写入 WordPress。

## 适用场景

- 采集工具站、产品库、资讯列表、目录站等页面内容
- 将外部页面批量整理后写入 WordPress 文章、自定义文章类型或自定义字段
- 为不同站点保存不同采集模板，后续重复使用
- 在大批量采集时，通过后台队列持续执行，减少单次请求超时风险

## 核心能力

- 输入列表页 URL，由 AI 辅助发现可采集的详情页 URL
- 支持直接分析详情页并抽取标题、摘要、网址、正文和自定义字段
- 支持保存可复用采集模板，后续直接创建队列任务
- 支持为不同任务选择不同 AI 模型
- 支持为标题、摘要、正文等字段单独控制是否进行 AI 重写
- 支持按目标文章类型、分类法和目录写入 WordPress
- 支持将正文以 Gutenberg 区块内容形式入库
- 支持写入 `links` 字段以及普通自定义字段
- 支持为入库文章写入作者、标签，以及 Yoast SEO 常用字段
- 支持暂停、恢复、取消、重试失败条目和删除已结束任务

## 后台流程

1. 在 `工具 -> AIditor` 中进入插件后台。
2. 在 `AI 设置` 中添加至少一个可用模型。
3. 在 `通用采集模板` 中填写来源 URL 和采集需求，生成并保存模板。
4. 在 `导入` 中选择模板、模型、目标文章类型和分类目录，创建队列任务。
5. 在 `任务列表` 和 `任务详情` 中查看进度、失败条目和最近处理结果。

## 支持的内容写入

- 文章标题
- 文章摘要
- 文章正文
- `links` 字段
- 普通自定义字段
- 目标分类法与目录
- 单个标签
- Yoast SEO 焦点关键词和元描述

## 队列与稳定性

- 采集任务通过后台队列执行，不依赖单个页面请求一次跑完
- 支持单轮多条连续处理、按时间片续跑和后台异步 worker
- 支持对临时 AI 故障、超时、429 和部分来源请求失败做重试
- 支持大批量采集时分轮推进，减少对站点前台访问的影响

## 环境要求

- WordPress 6.0+
- PHP 7.4+
- 可用的 OpenAI 兼容接口，或其他兼容 Chat Completions / Responses 风格的模型服务

## 安装方式

1. 将插件目录放入 `wp-content/plugins/aiditor`
2. 在 WordPress 后台启用 `AIditor`
3. 进入 `工具 -> AIditor`
4. 先完成 AI 模型配置，再开始创建采集模板和队列任务

## 适配说明

- 支持 WordPress 默认文章与自定义文章类型
- 支持层级分类法与非层级分类法
- 支持 ACF 的 `links` 字段写入
- 支持 Yoast SEO / Yoast SEO Premium 常用元字段写入

## 注意事项

- 插件默认不会预置任何模型地址、模型名称或 API 密钥
- 采集结果质量与来源页面结构、模型能力和采集需求描述直接相关
- 建议先用少量条目验证模板，再扩大批量任务规模
- 如果你的网站存在缓存或安全拦截，建议先确认 REST 和后台异步请求可正常运行
