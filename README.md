# AIditor

AIditor 是一个 WordPress 后台插件。它可以用 AI 分析外部页面，抽取结构化内容，重写正文，并写入文章或自定义文章类型。

你可以从列表页批量采集内容，也可以从单个详情页采编文章。还可以直接输入创作说明，让 AI 生成文章草稿。

## 主要功能

### AI 采集

- 从列表页发现详情页 URL。
- 从详情页抽取标题、摘要、正文、网址、封面图等字段。
- 支持自定义字段 JSON。
- 支持保存采集模板，后续重复使用。
- 支持队列任务，适合批量采集。
- 支持暂停、恢复、取消和失败重试。

### AI 采编

- 直接输入详情页 URL。
- 抽取标题、发布时间、关键词、摘要、正文、作者等字段。
- 可选择字段进行 AI 重写。
- 重写时可选择公共文章风格。
- 正文重写会尽量保留原文长度。
- 英文缩写、产品名、技术名词等会被保护，避免被误改。
- 发布前可选择要写入的字段。

### AI 创作

- 输入创作说明，直接生成文章。
- 支持选择 AI 模型和文章风格。
- 支持自定义补充风格要求。
- 支持预览标题、摘要、关键词、正文和图片。
- 发布时可写入草稿、待审核、私密或已发布文章。
- 正文远程图片会尝试转存到本地媒体库。

### 公共设置

- 支持多个 OpenAI 兼容模型配置。
- 支持维护公共文章风格。
- 文章风格可用于 AI 采集、AI 采编和 AI 创作。
- 支持设置默认文章状态、队列并发数、轮询间隔和日志保留天数。

## 适用场景

- 资讯站、工具站、产品库、资源目录站。
- 将外部页面整理为 WordPress 文章。
- 为自定义文章类型批量补充内容。
- 用统一风格重写采集内容。
- 根据选题快速生成中文文章草稿。

## 环境要求

- WordPress 6.0 或更高版本。
- PHP 7.4 或更高版本。
- 一个可用的 OpenAI 兼容模型接口。
- WordPress REST API 可正常访问。

插件不会内置模型地址、模型名称或 API 密钥。启用后，需要你自行配置模型。

## 安装方式

1. 下载发布包 `AIditor-v0.0.3.zip`。
2. 在 WordPress 后台进入“插件 -> 安装插件”。
3. 上传 ZIP 文件并启用插件。
4. 进入左侧菜单“AIditor”。
5. 打开“设置”，先添加 AI 模型配置。

也可以手动安装：

1. 解压插件目录。
2. 将 `aiditor` 文件夹放入 `wp-content/plugins/`。
3. 在 WordPress 后台启用 `AIditor`。

## 基本使用流程

### 1. 配置模型

进入 `AIditor -> 设置 -> AI 设置`。

新增一个模型配置，填写：

- 配置名称
- 接口地址
- API 密钥
- 模型名称
- 温度
- 最大 Tokens
- 请求超时

保存后，插件会隐藏密钥明文。后续任务只会显示密钥已配置。

### 2. 配置文章风格

进入 `AIditor -> 设置 -> 文章风格`。

你可以直接填写风格提示词，也可以先填写风格名称和需求，再调用 AI 生成风格。

保存后的风格会出现在：

- AI 采集的默认文章风格设置
- AI 采编的重写风格选择
- AI 创作的预设风格选择

### 3. 创建采集模板

进入 `AIditor -> AI采集 -> 通用采集模板`。

填写列表页 URL 和采集需求。插件会先分析页面，再识别详情页。你可以抽取首个详情页样例，并确认字段结果。

确认无误后，保存为模板。

### 4. 创建队列任务

进入 `AIditor -> AI采集 -> 导入`。

选择采集模板、模型、来源 URL、采集数量、文章类型、分类法和目录。建议先点击“预览”，确认候选内容后再创建任务。

任务创建后，可在任务列表查看进度。失败条目可以重试。

### 5. 使用 AI 采编

进入 `AIditor -> AI采编 -> AI采编`。

填写详情页 URL，选择模型和文章风格。先点击“开始采集”，再勾选需要重写的字段。重写完成后，选择发布字段和字段映射，即可写入 WordPress。

适合单篇内容精修、测试模板效果，或处理不需要队列的文章。

### 6. 使用 AI 创作

进入 `AIditor -> AI采编 -> AI创作`。

选择模型和风格，填写创作说明。生成后可预览文章内容，再设置文章类型、分类法、目录和字段映射。

发布时，插件会尝试把正文中的远程图片下载到媒体库，并重写图片地址。

## 支持写入的内容

- 文章标题
- 文章摘要
- 文章正文
- 文章状态
- 文章作者
- 目标分类法与目录
- 标签等非层级分类法
- 普通自定义字段
- ACF `links` 字段
- Yoast SEO 焦点关键词
- Yoast SEO 元描述

正文会尽量转换为 Gutenberg 可编辑内容。

## 字段映射

AI 采编和 AI 创作都支持字段映射。你可以决定某个字段写入哪里：

- WordPress Core 字段，例如标题、摘要、正文。
- Meta 字段，例如来源作者、来源链接。
- Taxonomy 字段，例如文章标签。

如果关键词字段映射到 `post_tag`，发布时会优先使用关键词作为标签。

## 安全与隐私

- 插件不会自带任何 API 密钥。
- 模型密钥保存在 WordPress 选项中。
- 后台公开设置不会返回密钥明文。
- REST 接口需要管理员权限。
- 发布包不包含测试目录、文档目录、构建目录、`.env`、`.git` 或本地开发配置。

请只采集你有权使用的内容。采集、改写和发布结果需要由站点管理员自行审核。

## 注意事项

- AI 输出质量取决于模型能力、来源页面结构和提示词质量。
- 建议先用少量内容测试，再扩大采集数量。
- 远程图片转存可能受目标站防盗链、超时或文件格式影响。
- 如果站点启用了安全插件或缓存，请确认 REST API 和后台异步请求可用。
- AI 生成内容可能有事实偏差，发布前请人工检查。
